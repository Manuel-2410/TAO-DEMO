10. Definiciones

- Paciente: Persona que acude a TAO buscando una solucion a sus sintomas, situacion o problemas, tanto fisicos, como emocionalmente.

- Tratamiento: Servicio que se le brinda al paciente.

- Lead: prospecto que mando un mensaje o realizo una llamada para pedir informacion.

- Agenda: Herramienta o registro para planificar y organizar actividades, citas o tareas en un calendario.

- Seguimiento: Proceso de monitorear el avance de actividades, proyectos o pacientes, asegurando que se cumplan objetivos.

- Diagnóstico: Evaluación sistemática para identificar un problema, ya sea médico, técnico o empresarial, con base en datos y análisis.

- ERP: (Enterprise Resource Planning): Software que integra procesos internos como contabilidad, inventarios, compras, producción y recursos humanos. Ejemplo: SAP, Oracle NetSuite.

- CRM: (Customer Relationship Management): Sistema para gestionar la relación con clientes: ventas, marketing, soporte y comunicación. Ejemplo: HubSpot, Salesforce.

- HIS: (Hospital Information System): Plataforma que centraliza la información clínica y administrativa de un hospital: agendas médicas, expedientes electrónicos, facturación y gestión de personal.

Diferencia entre CRM, ERP y EMR.

Sistema	                                     Enfoque	                                Ejemplo de uso
CRM	                            Relación con clientes	                    Registrar leads, campañas de marketing, historial de contacto
ERP	                            Procesos internos	                        Facturación, inventarios, contabilidad
EMR                             (Electronic Medical Record)	                Información clínica	Historia médica electrónica de pacientes


Ciclo de vida del desarrollo de software (SDLC)
- El SDLC es un marco que describe las fases para crear software:

- Planificación – Definir objetivos y alcance.

- Análisis de requisitos – Identificar necesidades del usuario.

- Diseño – Arquitectura y especificaciones técnicas.

- Implementación – Programación y construcción.

- Pruebas – Verificación y validación.

- Despliegue – Entrega al usuario final.

- Mantenimiento – Correcciones y mejoras continuas.

IEEE 830 (Especificación de Requisitos de Software).

El estándar IEEE 830 define cómo documentar requisitos de software de manera clara y verificable:

- Propósito: Evitar ambigüedades y asegurar trazabilidad.

- Contenido típico: Introducción, descripción general, requisitos funcionales, no funcionales, interfaces y restricciones.

- Beneficio: Mejora la comunicación entre clientes, desarrolladores y testers.

## Funciones fuera del alcance inicial
Las siguientes funciones no forman parte de la primera versión empresarial:

- Inteligencia artificial clínica.
- Diagnósticos médicos automáticos.
- Facturación fiscal completa.
- Integración con aseguradoras.
- Telemedicina.
- Chat clínico en tiempo real.
- Inventario avanzado.
- Comercialización para clínicas externas.
